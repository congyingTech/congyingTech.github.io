Ubuntu+GitHub+hexo搭建自己的专属博客

发表于 2016-04-03   |   分类于 博客环境搭建   |   | 阅读次数
前言:本教程基于ubuntu系统,用git pages托管博客,hexo又包含了各种各样的themes,简直是神方便!!!你需要有一个github账号,并在github账号下new一个repositories,名称必须是username.github.io,因为我们是免费的用户,类型是public,而且一个账户下面只能有一个io这种类型的repositories.关于github的创建并非本文的重点,如果有问题请移步google,自行搜索.

Hexo安装准备工作

  • 1）github: 你要有一个github的账号，并且在账号下new一个username.github.io的repositories.

  • 2) ubuntu上安装本地环境：

    1. 安装git：用Git做版本管理的孩纸，想必一定安装了git，Ubuntu下安装git，非常的方便：
      $sudo apt-get install git-core

    2. 安装Node.js:首先解释一下为什么要安装这个东东，因为hexo是依赖于Node.js和Git的。Node.js是什么？ wiki上的解释是：Node.js是一个开放源代码、跨平台的、可用于服务器端和网络应用的运行环境，他是用JavaScript语言编写的。官网上推荐的最佳方式是使用nvm：
      nvm的github主页：https://github.com/creationix/nvm
      安装nvm可以使用下面两个命令的任意一个：
      cURL
      $curl https://raw.github.com/creationix/nvm/master/install.sh | sh
      wget
      $wget -qO- https://raw.github.com/creationix/nvm/master/install.sh | sh
      安装完之后重启终端，并执行下列命令：
      $nvm install 4
      至此，本地环境安装完毕！恭喜！

安装并初始化Hexo

以上所有的准备步骤完成之后，可以使用npm装hexo了。
$npm install -g hexo-cli
如果在安装的过程中遇到一些问题，请移步参考http://notes.maxwi.com/2014/02/22/first-post/

建站

安装完成hexo后，执行下面的命令，完成建站： 

1. `$hexo init <folder>`  
2. `cd <folder>`  
3. `npm install`

检验是否可以启动本地服务器进行预览：
$hexo generate —> $hexo server
打开http://localhost:4000/进行访问，如果页面正常打开，那么恭喜，本地的博客已经搭建完毕了，只要托管到git pages上即可。

配置git并发布

  • 1）config.yml文件配置：首先在刚刚你init hexo的下,找到_config.yml,这个文件基本上是我们配置站最常用的文件。
    在_config.yml找到deploy的位置进行配置。
    Attention:
    在你看到这篇教程之前，你可能看过很多教程，把deploy的type设置为：github，然而在hexo3.0中，type应该变为git，并且在配置之前运行下面命令： （参考：https://github.com/hexojs/hexo/issues/1040）
    $npm install hexo-deployer-git --save
    运行完此命令，我们就可以大胆的配置_config.yml文件啦！！！

    **deploy:  
type: git  
repository: https://github.com/username/username.github.io.git  
branch: master**

    （此处repository可从github上哪里https复制过来，很方便的啦~）

  • 2）设置git全局变量：
    $git config --global user.name "your name here"
    $git config --global user.email "your_email@xx.com"

  • 3）发布：
    至此，配置完毕，我们可以将本地的站部署到github上啦！
    $hexo deploy
    记得每次发布的步骤：
    1. 在本地生成：$hexo generate
    2. 在本地预览：$hexo server
    3. 部署到git：$hexo deploy

遇见过的bug总结（坚持更新）

最近一次deploy的时候，发现让输入username和password之后，开始了无限期的等待，没有任何报错，也没法部署成功。
解决方案：$hexo clean并且删除.deploy_git文件夹之后，再进行$hexo deploy就可以部署成功啦~撒花~

总结

开始的时候，觉得自己建一个博客要好难，现在看来，也不是一件难于上青天的事情，关键是要勇敢的去尝试，这篇文章也只是简单的搭建好一个平台，你还可以个性化的定制自己的博客主题，加一些自己的social链接啦，RSS啦，等等等等，这些都随你的心意咯。
